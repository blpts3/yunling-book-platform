# 云凌书驿平台 - 部署指南

## 📋 部署前准备

### 1. 创建Supabase项目

1. 访问 [Supabase官网](https://supabase.com)
2. 点击 "Start your project" → "New project"
3. 填写项目信息：
   - Project name: `yunling-book`
   - Database Password: 设置一个强密码（请保存好）
   - Region: 选择离用户最近的区域（如 `Singapore`）
4. 等待项目创建完成（约2分钟）

### 2. 获取Supabase凭证

1. 进入项目控制台
2. 点击左侧 "Settings" → "API"
3. 复制以下信息：
   - **Project URL**: `https://xxx.supabase.co`
   - **anon public key**: `eyJhbG...` (以eyJ开头的长字符串)
   - **service_role key**: `eyJhbG...` (保密，不要暴露给前端)

### 3. 执行数据库Schema

1. 在Supabase控制台，点击左侧 "SQL Editor"
2. 点击 "New query"
3. 打开项目中的 `supabase-schema.sql` 文件
4. 复制全部内容，粘贴到SQL编辑器
5. 点击 "Run" 执行

### 4. 配置环境变量

1. 在项目根目录创建 `.env` 文件：

```bash
# 复制.env.example为.env
cp .env.example .env
```

2. 编辑 `.env` 文件，填入Supabase凭证：

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key-here
VITE_ADMIN_EMAIL=你的管理员邮箱@gmail.com
```

### 5. 导入初始数据（可选）

如果需要导入模拟数据到数据库：

```bash
# 安装依赖
npm install dotenv tsx

# 添加service_role密钥到.env
echo "SUPABASE_SERVICE_ROLE_KEY=your-service-role-key" >> .env

# 运行迁移脚本
npx tsx scripts/seed-data.ts
```

## 🚀 部署到Netlify

### 方法1：通过Git仓库自动部署（推荐）

#### Step 1: 初始化Git仓库

```bash
cd swjtu-mentor-platform
git init
git add .
git commit -m "Initial commit"
```

#### Step 2: 推送到GitHub

1. 在GitHub创建新仓库 `yunling-book-platform`
2. 关联远程仓库：

```bash
git remote add origin https://github.com/你的用户名/yunling-book-platform.git
git branch -M main
git push -u origin main
```

#### Step 3: 在Netlify部署

1. 访问 [Netlify](https://netlify.com) 并登录
2. 点击 "Add new site" → "Import an existing project"
3. 选择 "GitHub" 并授权
4. 选择你的仓库 `yunling-book-platform`
5. 配置构建设置：
   - **Build command**: `npm run build`
   - **Publish directory**: `dist`
6. 点击 "Environment variables" 添加：
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
7. 点击 "Deploy site"

部署完成后，Netlify会提供一个类似 `https://xxx.netlify.app` 的域名。

### 方法2：手动部署

```bash
# 构建项目
npm run build

# 安装Netlify CLI
npm install -g netlify-cli

# 登录Netlify
netlify login

# 部署
netlify deploy --prod --dir=dist
```

## 🔐 配置管理员权限

### 设置管理员账户

1. 注册时使用您在 `.env` 中配置的邮箱
2. 在Supabase控制台手动修改该用户的role为'admin'：

```sql
-- 在SQL Editor中执行
UPDATE users SET role = 'admin' WHERE email = '你的管理员邮箱';
```

### 保护管理后台路由

在 `src/App.tsx` 中添加管理员权限检查：

```typescript
// 在Admin组件中添加
import { useEffect, useState } from 'react'
import { useNavigate } from 'react-router-dom'
import { auth } from '@/lib/supabase'

function Admin() {
  const navigate = useNavigate()
  const [authorized, setAuthorized] = useState(false)

  useEffect(() => {
    const checkAdmin = async () => {
      const isAdmin = await auth.isAdmin()
      if (!isAdmin) {
        navigate('/')
      } else {
        setAuthorized(true)
      }
    }
    checkAdmin()
  }, [navigate])

  if (!authorized) return <div>加载中...</div>
  
  // ... 原有Admin内容
}
```

## 🌐 自定义域名（可选）

### 在Netlify设置自定义域名

1. 购买域名（如通过阿里云、腾讯云）
2. 在Netlify控制台：
   - 进入 "Site settings" → "Domain management"
   - 点击 "Add custom domain"
   - 输入您的域名（如 `www.yunlingbook.com`）
3. 配置DNS记录：
   - 在域名服务商处添加CNAME记录：
     ```
     类型: CNAME
     主机记录: www
     记录值: xxx.netlify.app
     ```
4. 等待DNS生效（通常几分钟到几小时）

## 🔧 常见问题

### Q1: 部署后出现空白页面？

**A:** 检查浏览器控制台的错误信息，通常是环境变量未正确配置。

### Q2: 注册后无法登录？

**A:** Supabase默认需要邮箱验证。可以在Supabase控制台关闭此功能：
- Authentication → Settings → Email Auth
- 关闭 "Enable email confirmations"

### Q3: 如何批量导入真实社团数据？

**A:** 编辑 `scripts/seed-data.ts`，将真实的社团数据添加到数组中，然后运行脚本。

### Q4: 国内访问Netlify速度慢？

**A:** 考虑使用国内CDN或部署到Vercel（国内访问更快）。
